# HLOCK Specification v0.7.0

## 1. Overview
HLOCK v0.7.0 introduces **First-Class Supply Chain Provenance**, becoming the first lockfile format with native, binary-optimized support for SLSA (Supply-chain Levels for Software Artifacts) and Sigstore cryptographic attestations.

As regulatory frameworks (e.g., US Executive Order 14028, EU Cyber Resilience Act) mandate Software Bills of Materials (SBOMs) and build provenance, package managers can no longer rely solely on integrity hashes (which only prove a tarball wasn't tampered with during download). They must prove *who* built the artifact and *where* the source code came from. 

By leveraging HLOCK's binary payload architecture, v0.7.0 embeds this cryptographic provenance with minimal byte overhead—something fundamentally impractical in JSON/YAML lockfiles due to base64-in-JSON bloat and string escaping taxes.

**Breaking Change:** This is a payload schema bump. The version byte changes from `0x04` to `0x05`. HLOCK v0.7.0 parsers will intentionally reject v0.4.0 and older payloads, but SHOULD gracefully handle v0.5.0/v0.6.0 payloads (without provenance) for forward compatibility during ecosystem migration.

## 2. File Structure
Unchanged from v0.6.0.

## 3. The Binary Payload (v0.7 Schema)

### 3.1 Payload Layout

| Field | Type | Size | Description |
| :--- | :--- | :--- | :--- |
| `Version` | Byte | 1 byte | Payload schema version. **MUST be `0x05`** for v0.7.0. |
| `SourceIdx`| Varint | 1+ bytes | Index mapping to `@source <idx>`. |
| `Major` | Varint | 1-9 bytes | Package major version. |
| `Minor` | Varint | 1-9 bytes | Package minor version. |
| `Patch` | Varint | 1-9 bytes | Package patch version. |
| `HashCount`| Varint | 1+ bytes | Number of integrity hashes. |
| `Hashes` | Array | Variable| Array of extended hash structs (see 3.2). |
| `FeatureCount`| Varint | 1+ bytes | Number of strings in this package's local Feature Table. |
| `Features` | Array | Variable| Array of `(VarintLen, UTF-8 Bytes)`. |
| `DepCount`| Varint | 1+ bytes | Number of direct dependencies. |
| `Dependencies`| Array | Variable| Array of dependency structs (unchanged from v0.5). |
| `CRC32` | Bytes | 4 bytes | Standard CRC32 (IEEE 802.3) checksum of all preceding bytes, little-endian. |

### 3.2 Extended Hash Struct Layout

The hash struct is extended to include an optional attestation pointer.

| Field | Type | Size | Description |
| :--- | :--- | :--- | :--- |
| `AlgoId` | Byte | 1 byte | Integer representing the hash algorithm (unchanged). |
| `HashLen` | Byte | 1 byte | Length of the digest bytes. |
| `Digest` | Bytes | Variable| The raw hash digest bytes. |
| `AttestationType` | Byte | 1 byte | `0x00` (None), `0x01` (External), `0x02` (Inline SLSA). |
| `Attestation`| Payload | Variable| **Only present if `AttestationType != 0x00`**. See 3.3. |

### 3.3 Attestation Payloads

**If `AttestationType == 0x01` (External Bundle):**
| Field | Type | Size | Description |
| :--- | :--- | :--- | :--- |
| `BundleHash` | Bytes | 32 bytes | SHA-256 hash of the external Sigstore bundle file. The fetcher must locate this file alongside the lockfile or in a known remote cache. |

**If `AttestationType == 0x02` (Inline SLSA Predicate):**
A highly compressed, binary-only subset of a SLSA v1.0 predicate, avoiding JSON entirely.
| Field | Type | Size | Description |
| :--- | :--- | :--- | :--- |
| `BuilderLen` | Varint | 1+ bytes | Length of the builder identifier string. |
| `Builder` | Bytes | Variable| UTF-8 string identifying the builder (e.g., `github.com/my-org/.github/workflows/build.yml@refs/heads/main`). |
| `SourceLen` | Varint | 1+ bytes | Length of the source URI string. |
| `Source` | Bytes | Variable| UTF-8 string of the source code (e.g., `git+https://github.com/my-org/pkg@v1.2.3`). |

## 4. Public API Specification

### 4.1 Core Data Structures

```rust
#[derive(Debug, Clone, PartialEq, Eq)]
pub enum Attestation {
    None,
    ExternalBundleSha256([u8; 32]),
    InlineSlsa(SlsaPredicate),
}

#[derive(Debug, Clone, PartialEq, Eq)]
pub struct SlsaPredicate {
    pub builder: String,
    pub source: String,
}

#[derive(Debug, Clone, PartialEq, Eq)]
pub struct IntegrityHash {
    pub algo: HashAlgorithm,
    pub digest: Vec<u8>,
    pub attestation: Attestation, // NEW in v0.7.0
}
```

### 4.2 Error Handling Additions

```rust
pub enum Error {
    // ... existing errors ...
    
    #[error("Line {line_number}: Unsupported attestation type {type_id}")]
    UnknownAttestationType { line_number: usize, type_id: u8 },
}
```

### 4.3 Parser Behavior
*   `unpack_payload`: If `Version == 0x04`, parse exactly as v0.5/v0.6 (implicitly setting `attestation: Attestation::None`).
*   `unpack_payload`: If `Version == 0x05`, parse the new hash struct, reading the `AttestationType` byte and the corresponding attestation payload. If `AttestationType > 0x02`, return `UnknownAttestationType`.
*   `pack_payload`: Always serialize as `Version == 0x05`.

## 5. Implementation Notes for Package Managers
*   **Zero-Trust Fetching:** When installing a package, if `Attestation::InlineSlsa` is present, the package manager should verify that the `builder` matches an allowlist defined in the workspace configuration before extracting the tarball.
*   **SBOM Generation:** Because the provenance is tied directly to the specific `IntegrityHash` (which is tied to the specific `Content ID`), generating a CycloneDX or SPDX SBOM from an HLOCK file is a direct 1:1 mapping with zero guesswork.
