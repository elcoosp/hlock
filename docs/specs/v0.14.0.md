# HLOCK Specification v0.14.0

## 1. Overview

HLOCK v0.14.0 is **"The Supply Chain & Merge Intelligence Release"**. It solves the two most painful operational problems with lockfiles at scale: merge conflicts in monorepo branching workflows, and O(n) full-parse costs that make 100k+ package lockfiles impractical to query. It also brings compliance-ready SBOM generation, a declarative linting engine, and resolution provenance tracking.

This is a non-breaking release from v0.13. The payload version byte remains `0x08`. The binary payload format is unchanged. All new functionality is additive — new functions, new header directives, new types.

### Design Principles

- **Merge, Don't Re-resolve** — A lockfile merge must never invoke a resolver. It is a deterministic, structural 3-way merge over sorted package lists. When the merge algorithm cannot reconcile a conflict, it reports it precisely. The human decides.
- **Pay For What You Use** — A CI step that checks one package's hash should not parse 99,999 others. The lazy parser indexes line offsets in a single linear scan (no payload decoding), then decodes individual payloads on demand via binary search.
- **Compliance Without Scripts** — SBOM generation, lint rules, and provenance tracking are first-class library features, not post-processing scripts that drift out of sync with the format.
- **Governance as Code** — Lint rules are trait objects. Teams compose rule sets, share them across repos, and enforce them in CI. The library ships with a standard rule set covering the most common governance requirements.

## 3. Header Directives

### 3.1 New Directive: `@provenance`

```
@provenance <package> <constraint> <constrained_by> <dep_type_id> <source_type_id> <depth>
```

Records the resolution provenance for a package — why this specific version is in the lockfile.

| Field | Format | Description |
|---|---|---|
| `package` | String | Package name (must match a package line) |
| `constraint` | String | Semver constraint that was satisfied (e.g. `^4.17.0`) |
| `constrained_by` | String | Name of the package that imposed this constraint (empty string for direct dependencies) |
| `dep_type_id` | Decimal | Same IDs as payload: `0` = Runtime, `1` = Dev, `2` = Peer, `3` = Optional, `4` = OptionalTarget |
| `source_type_id` | Decimal | `0` = Registry, `1` = Local, `2` = Git, `3` = Workspace, `4` = CasHttp, `5` = Ipfs |
| `depth` | Decimal | Dependency depth: `0` = direct, `1` = dependency of direct, etc. |

**Rules:**
- At most one `@provenance` directive per package name. If duplicates exist, the last one wins.
- A `@provenance` directive referencing a package name not present in the lockfile is ignored by parsers but should produce a lint warning.
- `depth` is informational. It does not affect resolution or installation.

**Example:**
```
@provenance lodash ^4.17.0 app 0 0 2
@provenance react ^18.0.0 app 0 0 1
@provenance jest ^29.0.0 app 1 0 1
@provenance pretty-format ^29.0.0 jest 0 0 2
```

### 3.2 Unchanged Directives

All v0.13 directives are unchanged: `@source`, `@override`, `@feature`, `@workspace-root`, `@workspace-pkg`, `@hoist-boundary`, `@metadata`, `@artifact`, `@patch`, `@digest`, `@signature`.

## 6. Public API Specification

### 6.1 Three-Way Lockfile Merge

```rust
#[derive(Debug, Clone, PartialEq, Eq)]
pub struct MergeConflict {
    pub package_name: String,
    pub base: Option<Package>,
    pub ours: Package,
    pub theirs: Package,
}

#[derive(Debug, Clone, Copy, PartialEq, Eq)]
pub enum ConflictStrategy {
    Ours,
    Theirs,
    Fail,
}

#[derive(Debug, Clone, PartialEq, Eq)]
pub struct MergeResult {
    pub lockfile: Lockfile,
    pub conflicts: Vec<MergeConflict>,
}

pub fn merge_lockfiles(
    base: &Lockfile,
    ours: &Lockfile,
    theirs: &Lockfile,
    strategy: ConflictStrategy,
) -> Result<MergeResult, Error>
```

**Merge algorithm:**

1. **Sources**: Merge source lists. If both branches added a source at the same index with different URLs, that is a conflict. New sources from either branch are appended with renumbered indices.
2. **Packages**: Three-way merge over the sorted package list:
   - Present in base, unchanged in both → keep
   - Present in base, altered in ours only → take ours
   - Present in base, altered in theirs only → take theirs
   - Present in base, altered in both → if identical, keep; otherwise conflict
   - Absent in base, added in ours only → include, remap source index
   - Absent in base, added in theirs only → include, remap source index
   - Absent in base, added in both → if identical, keep; otherwise conflict
   - Present in base, removed in ours → remove
   - Present in base, removed in theirs → remove
   - Present in base, removed in one and altered in other → conflict
3. **Overrides**: Take union. If both branches override the same package, take the more restrictive version (or conflict if different).
4. **Features**: Take union.
5. **Metadata**: Take union.
6. **Artifacts/Patches**: Take union.
7. **Provenance**: Re-derive after merge (or take union and remove entries for removed packages).
8. **Hoist boundaries**: Take union.

**Conflict resolution** depends on `strategy`:
- `Fail`: Return an `Error` containing all conflicts.
- `Ours`: Take `ours` version for every conflict.
- `Theirs`: Take `theirs` version for every conflict.

After merge, the result is re-serialized to ensure consistent source index mapping and sorted package order.

### 6.2 Lazy / Zero-Copy Parsing

```rust
#[derive(Debug, Clone)]
pub struct LazyLockfile {
    /* private fields */
}

#[derive(Debug, Clone, PartialEq, Eq)]
pub struct LockfileHeader {
    pub sources: Vec<Source>,
    pub overrides: Vec<Override>,
    pub features: Vec<(String, Vec<String>)>,
    pub metadata: Vec<(String, String)>,
    pub workspace_root: Option<String>,
    pub workspace_pkgs: Vec<WorkspacePkg>,
    pub hoist_boundaries: Vec<HoistBoundary>,
    pub artifacts: Vec<ArtifactDirective>,
    pub patches: Vec<PatchDirective>,
}

impl LazyLockfile {
    /// Scan the lockfile content, parsing only the header and indexing
    /// package line offsets. No binary payloads are decoded.
    pub fn scan(content: &str) -> Result<LazyLockfile, Error>

    /// Number of packages in the lockfile (O(1)).
    pub fn package_count(&self) -> usize

    /// Iterator over package names in sorted order (O(1) per iteration).
    pub fn package_names(&self) -> impl Iterator<Item = &str>

    /// Parse and return a single package by name (O(log n) + single payload decode).
    pub fn get_package(&self, name: &str) -> Result<Option<Package>, Error>

    /// Parse and return all packages from a given source index.
    pub fn get_packages_by_source(&self, source_idx: usize) -> Result<Vec<Package>, Error>

    /// Parse and return all packages whose names match a predicate.
    pub fn get_packages_where(
        &self,
        predicate: impl Fn(&str) -> bool,
    ) -> Result<Vec<Package>, Error>

    /// Access the parsed header (sources, overrides, features, etc.).
    pub fn header(&self) -> &LockfileHeader

    /// Validate the @digest directive if present (without parsing payloads).
    pub fn validate_digest(&self) -> Result<(), Error>

    /// Convert to a full Lockfile by parsing all remaining payloads.
    pub fn into_full(self) -> Result<Lockfile, Error>
}
```

**Implementation constraints:**
- `scan()` must be O(n) in lines but must not decode any Base64URL payload data.
- Package index entries store: `(name: String, line_start: usize, line_end: usize)`.
- `get_package()` binary-searches the sorted index, then calls `unpack_payload()` on the single relevant line.
- `LazyLockfile` holds an `Arc<String>` (or `Arc<str>`) of the raw content for zero-copy name borrowing.

### 6.4 Lockfile Linting Engine

```rust
#[derive(Debug, Clone, PartialEq, Eq)]
pub enum LintSeverity {
    Error,
    Warning,
    Info,
}

#[derive(Debug, Clone, PartialEq, Eq)]
pub struct LintFinding {
    pub rule: String,
    pub severity: LintSeverity,
    pub package: Option<String>,
    pub message: String,
}

#[derive(Debug, Clone)]
pub struct LintReport {
    pub findings: Vec<LintFinding>,
}

impl LintReport {
    pub fn has_errors(&self) -> bool
    pub fn errors(&self) -> impl Iterator<Item = &LintFinding>
    pub fn warnings(&self) -> impl Iterator<Item = &LintFinding>
}

pub trait LintRule: Send + Sync {
    fn name(&self) -> &str;
    fn check(&self, lockfile: &Lockfile) -> Vec<LintFinding>;
}

pub fn lint(lockfile: &Lockfile, rules: &[Box<dyn LintRule>]) -> LintReport
```

**Built-in rules** (in `hlock::lint` module):

```rust
pub mod lint {
    pub struct NoGitUrls;        // Error: no Source::Git
    pub struct RequireIntegrity;  // Error: non-workspace packages need ≥1 hash
    pub struct NoSha1;           // Warning: SHA-1 is deprecated
    pub struct NoPeerAsRuntime;  // Warning: peer in runtime deps
    pub struct MaxDepth { pub max: u32 }; // Warning: depth exceeds threshold
    pub struct RequireAttestation; // Info: packages should carry attestations
}
```

**Custom rule example:**

```rust
struct NoInternalDeps;
impl LintRule for NoInternalDeps {
    fn name(&self) -> &str { "no-internal-deps" }
    fn check(&self, lockfile: &Lockfile) -> Vec<LintFinding> {
        lockfile.packages.iter()
            .filter(|p| p.name.starts_with("@internal/"))
            .map(|p| LintFinding {
                rule: "no-internal-deps".into(),
                severity: LintSeverity::Error,
                package: Some(p.name.clone()),
                message: format!("Internal package {} in production lockfile", p.name),
            })
            .collect()
    }
}
```

```rust
pub enum ProvenanceSourceType {

    pub fn dependency_chain(&self, package_name: &str) -> Vec<ResolutionProvenance>
```


**Serialization** (in `serialize`):
```
@provenance lodash ^4.17.0 app 0 0 2
```

Field encoding:
- `dep_type_id`: `0` = Runtime, `1` = Dev, `2` = Peer, `3` = Optional, `4` = OptionalTarget
- `source_type_id`: `0` = Registry, `1` = Local, `2` = Git, `3` = Workspace, `4` = CasHttp, `5` = Ipfs
- `depth`: decimal integer

**Deserialization** (in `deserialize`):
`@provenance` lines after `@patch` lines and before `@digest`. Unknown lines are skipped.
