# hlock v0.18.0 Release Research & Specification

Based on deep analysis of the current supply-chain security ecosystem, OpenSSF initiatives, and competitive landscape, here's what v0.18.0 should target:

---

## Market Research Summary

### Current Landscape (2024-2025)

**Supply chain attacks have increased 742% since 2019** (Sonatype State of the Software Supply Chain 2024). Key developments driving tool requirements:

1. **OpenSSF Scorecard** now scores 1M+ repos — users want lockfile tools that consume Scorecard data
2. **OSV** (osv.dev) has become the de facto open vulnerability API — free, no auth, covers 150+ ecosystems
3. **Sigstore** graduated CNCF incubation — cosign/Rekor are production-ready for package signing verification
4. **SLSA v1.0** is stable — CI systems now produce SLSA provenance natively (GitHub Actions, GitLab)
5. **CycloneDX 1.6** added VEX 1.1 support and service catalog
6. **GUAC** (Graph for Understanding Artifact Composition) hit v0.5 — supply chain graph queries are now feasible
7. **npm audit** added `--fix` and `--json` — users expect remediation, not just detection
8. **Socket.dev** popularized "dependency risk scoring" — license + maintainer trust + typo detection
9. **SEC SBOM requirements** (March 2026 deadline) driving SBOM quality tooling
10. **EU Cyber Resilience Act** (2027 enforcement) mandates SBOM + vulnerability disclosure

### Key Gaps for hlock v0.18.0

1. **No live vulnerability data** — advisories are embedded at lockfile creation time, not queried at audit time
2. **No auto-remediation** — users must manually update packages
3. **No import path** — can't adopt hlock without rewriting lockfiles
4. **No config file** — every policy must be passed via CLI flags
5. **No Scorecard integration** — can't factor in upstream project health
6. **No Sigstore verification** — can't verify cosign-signed artifacts
7. **No outdated checking** — users don't know what's stale

## Feature Specifications

### G01: `hlock outdated`

**Purpose:** Check for newer versions of packages in the lockfile by querying registry metadata.

```
hlock outdated [OPTIONS] <FILE>

Arguments:
  <FILE>  Path to lockfile (- for stdin)

Options:
      --major           Show major version updates (default: minor/patch only)
      --dep-type <TYPE>  Filter: runtime, dev, peer, all [default: all]
      --format <FORMAT>  Output format: text, json [default: text]
      --timeout <SECS>   HTTP request timeout [default: 30]
  -h, --help            Print help
```

**Implementation:**

Query the npm registry for each package's `dist-tags.latest`:

```
GET https://registry.npmjs.org/<package>

Response: { "dist-tags": { "latest": "4.17.22", "next": "5.0.0-beta.1" }, "versions": { ... } }
```

Or use the abbreviated endpoint:

```
GET https://registry.npmjs.org/<package>/latest

Response: { "version": "4.17.22", ... }
```

**Rate limiting:** npm registry allows ~1000 req/min for unauthenticated requests. For lockfiles with 1000+ packages, batch 50 concurrent requests with `reqwest` async.

**Text output:**

```

─────────   ───────   ──────    ─────
lodash      4.17.21   4.17.22   patch
react       18.3.1    19.0.0    major

old-dep     1.0.0     2.0.0     major
pkg-git     2.0.0     —         git (skipped)
my-app      1.0.0     —         workspace (skipped)

4 outdated (2 major, 0 minor, 2 patch)
```

**JSON output:**

```json

### G02: `hlock audit --online`

**Purpose:** Enhance audit with live vulnerability data from OSV.

**OSV API:**

```
POST https://api.osv.dev/v1/query

Body: {
  "package": { "name": "lodash", "ecosystem": "npm" },
  "version": "4.17.21"
}

Response: {
  "vulns": [
    {
      "id": "GHSA-jf85-cq4p-4qr8",
      "summary": "Command Injection in lodash",
      "severity": [{ "type": "CVSS_V3", "score": "7.2" }],
      "references": [{ "type": "ADVISORY", "url": "https://github.com/advisories/GHSA-jf85-cq4p-4qr8" }],
      "affected": [{ "package": { "name": "lodash", "ecosystem": "npm" }, "ranges": [{ "type": "SEMVER", "events": [{ "introduced": "0" }, { "fixed": "4.17.21" }] }] }]
    }
  ]
}
```

**Behavior:**

```bash
# Current: offline only (embedded advisories)
hlock audit demo.hlock

# New: online mode queries OSV for each package
hlock audit demo.hlock --online

# New: combine offline + online
hlock audit demo.hlock --online --merge
```

**`--merge` strategy:**

1. Start with embedded advisories (offline)
2. Query OSV for each package
3. Merge: if OSV reports a vuln not in embedded advisories, add it
4. If OSV reports a vuln IS in embedded advisories, prefer the embedded one (it may have VEX status)
5. Deduplicate by advisory ID

**Text output (online mode):**

```
CRITICAL  old-dep     CVE-2024-0001   *                  (embedded)
MEDIUM    express     GHSA-2024-abcd  >=4.0.0 <4.22.0   (embedded, VEX: not_affected)
HIGH      lodash      GHSA-jf85-cq4p  <4.17.21          (online)

---
Total: 2 critical/high, 1 medium, 0 low, 0 info
Sources: 2 embedded, 1 online
```

**Rate limiting:** OSV allows unauthenticated queries at ~1000/min. Batch 50 concurrent.

### G03: `hlock fix`

**Purpose:** Auto-remediate vulnerabilities by updating packages to fixed versions.

```
hlock fix [OPTIONS] <FILE>

Arguments:
  <FILE>  Path to lockfile (must be a real file, not stdin)

Options:
      --audit           Fix audit findings (vulnerabilities)
      --outdated         Fix outdated packages
      --all              Fix both audit and outdated
      --dry-run          Show what would change without modifying
      --interactive      Prompt for each fix
      --format <FORMAT>  Output format: text, json [default: text]
  -h, --help            Print help
```

**Implementation:**

For each vulnerable/outdated package:

1. Query OSV/registry for the fixed version
2. Update the package in the lockfile (major, minor, patch)
3. Re-serialize the lockfile
4. Re-validate digest

**`--dry-run` output:**

```
Would fix 2 vulnerabilities:

Package   Current   Fixed     Advisory
────────  ───────   ─────     ────────
lodash    4.17.21   4.17.22   GHSA-jf85-cq4p-4qr8
old-dep   1.0.0     2.0.0     CVE-2024-0001

Run without --dry-run to apply.
```

**`--interactive` output:**

```
Fix lodash 4.17.21 → 4.17.22? [y/n/a/q] y
  Fixed GHSA-jf85-cq4p-4qr8 (prototype pollution)

Fix old-dep 1.0.0 → 2.0.0? [y/n/a/q] n
  Skipped CVE-2024-0001 (major update)

Applied 1 fix. Re-serialize and re-validate...
✓ Digest updated.
```

**Exit codes:**

| Code | Meaning |
|------|---------|
| 0 | Fixes applied (or --dry-run with fixes available) |
| 1 | No fixes needed |
| 2 | Error |

### G04: `hlock import`

**Purpose:** Convert existing lockfiles to hlock format.

```
hlock import [OPTIONS] <SOURCE>

Arguments:
  <SOURCE>  Source lockfile path

Options:
      --from <FORMAT>  Input format: yarn, npm, pnpm [required]
      --output <FILE>  Output path [default: stdout]
      --registry <URL> Default registry URL [default: https://registry.npmjs.org/]
  -h, --help          Print help
```

**yarn.lock parsing:**

```yaml
"lodash@^4.17.0":
  version "4.17.21"
  resolved "https://registry.yarnpkg.com/lodash/-/lodash-4.17.21.tgz#679b1..."
  integrity sha512-abc123...
  dependencies:
    something "^1.0.0"
```

**package-lock.json parsing:**

```json
{
  "packages": {
    "node_modules/lodash": {
      "version": "4.17.21",
      "resolved": "https://registry.npmjs.org/lodash/-/lodash-4.17.21.tgz",
      "integrity": "sha512-abc123...",
      "requires": {}
    }
  }
}
```

**Implementation:**

1. Parse source lockfile format
2. Extract package name, version, resolved URL, integrity hash, dependencies
3. Map integrity hashes: `sha512-abc...` → `IntegrityHash { algo: Sha512, digest: ... }`
4. Build dependency graph
5. Construct `Lockfile` struct
6. Serialize via `serialize()`

**Text output:**

```
Imported 6 packages from yarn.lock (v1)
Sources: 1 (https://registry.yarnpkg.com/)
Warnings:
  - pkg-git: git dependency, included as-is
  - my-app: workspace package, included as workspace
```

**JSON output:**

```json
{
  "imported": 6,
  "skipped": 0,
  "warnings": 2,
  "source_format": "yarn-v1",
  "source_file": "yarn.lock"
}
```

### G05: `.hlockrc` Config File

**Purpose:** Persist policies, trusted keys, and default settings.

**Search order:**

1. `--config <PATH>` (CLI flag)
2. `.hlockrc` in current directory
3. `.hlockrc` in project root (walk up to `.git`)
4. `~/.config/hlock/config.toml` (XDG)
5. Built-in defaults

**Format (TOML):**

```toml
[audit]
online = true
severity = "medium"

[lint]
rules = ["no-git-urls", "require-integrity"]
exclude-rules = ["deny-postinstall"]

[verify]
trusted-keys = [
  "ci@company.com:ed25519:9d61b19deffd5a60ba844af492ec2cc44449c5697b326919703bac031cae7f60",
]

[licenses]
allow = ["MIT", "Apache-2.0", "BSD-2-Clause", "BSD-3-Clause", "ISC"]
deny = ["GPL-3.0", "AGPL-3.0"]
strict = true

[outdated]
check_major = false
timeout = 30

[import]
default_registry = "https://registry.npmjs.org/"

[policy]
deny-hooks = ["postinstall"]
build-env = "node>=20.11.0"

[output]
color = "auto"
format = "text"
quiet = false
verbose = false
```

**Implementation:**

```rust
#[derive(Deserialize, Default)]
#[serde(default)]
struct HlockConfig {
    audit: AuditConfig,
    lint: LintConfig,
    verify: VerifyConfig,
    licenses: LicensesConfig,
    outdated: OutdatedConfig,
    import: ImportConfig,
    policy: PolicyConfig,
    output: OutputConfig,
}
```

**Precedence:** CLI flags > env vars > project `.hlockrc` > global config > defaults.

### G06: `hlock explain`

**Purpose:** Explain a finding (lint, audit, or check result) in detail.

```
hlock explain <RULE_OR_ID>

Arguments:
  <RULE_OR_ID>  Rule name (e.g., "no-git-urls") or advisory ID (e.g., "CVE-2024-0001")

Options:
      --format <FORMAT>  Output format: text, json [default: text]
  -h, --help            Print help
```

**Text output (lint rule):**

```
Rule: no-git-urls
Severity: error
Category: supply-chain

Description:
  Packages resolved from git URLs are non-reproducible and bypass
  registry integrity checks. Git repositories can be force-pushed,
  making builds non-deterministic.

Why it matters:
  - Git commits can be rewritten (force push)
  - No integrity guarantee (no hash verification)
  - Network-dependent at build time
  - Violates SLSA Level 1 requirements

How to fix:
  - Use a registry version instead: lodash@4.17.21
  - If using a fork, publish to a private registry
  - If using a specific commit, use git+https://...#<commit>

Related rules:
  - require-integrity
  - require-attestation

References:
  - https://slsa.dev/spec/v1.0/requirements
  - https://github.com/ossf/scorecard/blob/main/docs/checks.md

Example:
  # Before (fails)
  @source 0 git+https://github.com/pkg.git
  lodash  <base64>
  
  # After (passes)
  @source 0 https://registry.npmjs.org/
  lodash  <base64>
```

**Text output (advisory):**

```
Advisory: CVE-2024-0001
Severity: critical
Package: old-dep (1.0.0)
Affected: *

Description:
  Arbitrary code execution via prototype pollution in old-dep.
  An attacker can inject properties into Object.prototype,
  leading to privilege escalation.

How to fix:
  Update old-dep to version 2.0.0 or later.
  Run: hlock fix --audit demo.hlock

References:
  - https://github.com/advisories/CVE-2024-0001
