# HLOCK Specification v0.3.0

## 1. Overview
HLOCK v0.3.0 introduces the concepts of **Package Sources**, **Dependency Profiles**, and **Version Overrides**. As lockfiles scale to support monorepos, private registries, and complex environment builds (e.g., dev vs. prod), simply listing a package name and a hash is no longer sufficient. 

v0.3.0 solves this by introducing a **File Header** for global deduplication of registries and overrides, and expanding the binary payload to categorize dependencies and link them to specific sources.

**Breaking Change:** HLOCK v0.3.0 parsers will intentionally reject v0.2.0 and v0.1.0 payloads due to schema incompatibilities. There is no backward compatibility.

## 2. File Structure
The file is UTF-8 encoded, LF line endings (`\n`). It is split into two distinct sections separated by a single blank line: a **Metadata Header** and a **Package Graph**.

```text
<header directives>
<empty line>
<package lines>
```

### 2.1 The Metadata Header
The header contains global metadata required to interpret the package graph. It must appear at the very top of the file. Directives are prefixed with `@`.

*   `@source <idx> <uri>`: Defines a package source (e.g., a registry URL, local path, or git remote). `<idx>` is a 0-based integer. Packages will reference this index to know where to fetch the artifact.
*   `@override <name> <from_version> -> <to_version>`: Forces the dependency resolver to substitute `<to_version>` whenever `<name>` at `<from_version>` is requested.

**Example Header:**
```text
@source 0 https://registry.npmjs.org/
@source 1 https://packages.my-company.com/
@override react 18.2.0 -> 18.2.1
```

### 2.2 The Package Graph
Unchanged in concept from v0.2.0. One package per line, strictly tab-delimited.

```text
<package_name>\t<payload_base64url>
```

## 3. The Binary Payload (v0.3 Schema)
The binary payload (before Base64URL encoding) has been expanded to support sources and dependency typing. All multi-byte integers remain Unsigned LEB128 (Varint).

### 3.1 Payload Layout

| Field | Type | Size | Description |
| :--- | :--- | :--- | :--- |
| `Version` | Byte | 1 byte | Payload schema version. **MUST be `0x02`** for v0.3.0. |
| `SourceIdx`| Varint | 1+ bytes | Index of the download source mapping to `@source <idx>` in the header. |
| `Major` | Varint | 1-9 bytes | Package major version. |
| `Minor` | Varint | 1-9 bytes | Package minor version. |
| `Patch` | Varint | 1-9 bytes | Package patch version. |
| `HashLen` | Byte | 1 byte | Length of the integrity hash in bytes. |
| `Hash` | Bytes | `HashLen` bytes | The raw integrity hash bytes. |
| `DepCount`| Varint | 1+ bytes | Number of direct dependencies. |
| `Dependencies`| `DepCount` x Struct | Variable| Array of dependency structs (see 3.2). |
| `CRC32` | Bytes | 4 bytes | Standard CRC32 (IEEE 802.3) checksum of all preceding bytes, little-endian. |

### 3.2 Dependency Struct Layout
Instead of a simple Varint index, each dependency is now a struct containing an index and a profile type.

| Field | Type | Size | Description |
| :--- | :--- | :--- | :--- |
| `LineIdx` | Varint | 1+ bytes | 0-based line index of the dependency in the sorted package graph (relative to the start of the package section). |
| `DepType` | Byte | 1 byte | Integer representing the dependency profile (see 3.3). |

### 3.3 Dependency Type Enum
*   `0x00`: **Runtime** (Required for production execution)
*   `0x01`: **Dev** (Required for development/testing only)
*   `0x02`: **Peer** (Expected to be provided by the consumer; not fetched directly)
*   `0x03`: **Optional** (Fetched only if specific conditions are met)

## 4. Public API Specification

### 4.1 Core Data Structures

```rust
pub enum Source {
    Registry(String), // e.g., "https://registry.npmjs.org/"
    Local(String),    // e.g., "file:///path/to/local/pkg"
    Git(String),      // e.g., "https://github.com/user/repo.git"
}

pub enum DepType {
    Runtime,
    Dev,
    Peer,
    Optional,
}

pub struct Dependency {
    pub name: String,
    pub dep_type: DepType,
}

pub struct Override {
    pub name: String,
    pub from_version: String,
    pub to_version: String,
}

pub struct Package {
    pub name: String,
    pub source_idx: usize,
    pub major: u64,
    pub minor: u64,
    pub patch: u64,
    pub hash: Vec<u8>,
    pub dependencies: Vec<Dependency>,
}

pub struct Lockfile {
    pub sources: Vec<Source>,
    pub overrides: Vec<Override>,
    pub packages: Vec<Package>,
}
```

### 4.2 Error Handling Additions

```rust
pub enum Error {
    // ... existing v0.2 errors ...
    
    #[error("Line {line_number}: Package references undefined source index {index}")]
    MissingSource { line_number: usize, index: usize },

    #[error("Invalid header syntax at line {line_number}: {reason}")]
    InvalidHeader { line_number: usize, reason: String },

    #[error("Line {line_number}: Unsupported dependency type {type_id}")]
    UnknownDepType { line_number: usize, type_id: u8 },
}
```

### 4.3 Core Functions

The API shifts from serializing a raw `Vec<Package>` to serializing a unified `Lockfile` struct.

*   `fn serialize(lockfile: &mut Lockfile) -> Result<String, Error>`
    *   Validates that all `source_idx` values in `lockfile.packages` exist in `lockfile.sources`.
    *   Writes `@source` directives.
    *   Writes `@override` directives.
    *   Writes an empty line separator.
    *   Sorts `lockfile.packages` alphabetically.
    *   Writes package lines using the v0.3 binary payload spec.
*   `fn deserialize(content: &str) -> Result<Lockfile, Error>`
    *   Parses lines until an empty line is hit, populating `Lockfile.sources` and `Lockfile.overrides`.
    *   Parses remaining lines as packages using v0.3 payload unpacking.
    *   Maps the binary `LineIdx` + `DepType` back to resolved `Dependency` structs containing the actual package name and `DepType` enum.
