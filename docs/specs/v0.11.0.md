# HLOCK Specification v0.11.0

## 1. Overview
HLOCK v0.11.0 is **"The Interface & Artifact Transparency Release"**. It introduces language-agnostic mechanisms to lock module interfaces (preventing dependency confusion attacks) and lock dynamically fetched or compiled artifacts (preventing binary supply chain attacks). 

**Crucially, v0.11 is a hard breaking change from all previous versions.** Because the project is in alpha, all backward-compatibility shims, version-sniffing heuristics, and `CompatMode` enums have been completely removed. The binary format is now strictly ordered, requiring zero guessing from the parser.

### Design Principles
- **Language Agnostic** – Removes all assumptions about specific ecosystems (e.g., no hardcoded `node_modules`, no JS-specific lifecycle script names).
- **Strict Schema** – The binary payload is a strictly sequential list of fields. Parsers do not check remaining byte lengths to guess the version; they read the `0x07` version byte and expect the v0.11 layout exactly.
- **Interface Integrity** – Locks the public interface map (exports) of a module, ensuring dependency confusion attacks are cryptographically detectable.
- **Artifact Integrity** – Locks the hashes of dynamically generated or fetched platform-specific binaries, closing the "trusted post-link script" loophole.

## 2. File Structure

```text
<header directives>
<empty line>
<package lines>
<optional artifact directives>
<optional patch directives>
<optional signature directive>
```

Structural changes from v0.10:
1. New `@artifact` directives appear after package lines.
2. New generic `@metadata` directives can appear in the header.

## 3. Header Directives

### 3.1 Core Directives (Unchanged)
```text
@source <idx> <uri>
@override <name> <ver> -> <ver>
@feature <name> [flag1,flag2,...]
```

### 3.2 Workspace Directives (Unchanged)
```text
@workspace-root <path>
@workspace-pkg <name> <manifest_path>
@hoist-boundary <consumer> -> [dep1,dep2,...]
```

### 3.3 Metadata Directive (New)
```text
@metadata <key> <value>
```
Stores arbitrary key-value pairs associated with the lockfile or specific packages. Used primarily for SBOM (Software Bill of Materials) generation. Standard keys include `license`, `author`, `homepage`, and `repository`.

**Rules:**
- Can appear multiple times.
- If a key appears multiple times, the last one wins.

### 3.4 Artifact Directive (New)
```text
@artifact <content_id_hex> <os_id> <arch_id> <relative_path>
```
Declares that a platform-specific artifact (e.g., a compiled binary, a WASM module, a dynamically fetched archive) is associated with a locked package.

| Field | Description |
|---|---|
| `content_id_hex` | 16-character lowercase hex FNV-1a hash of the package. |
| `os_id` | Target OS ID (e.g., `0x01` for Linux, `0x02` for MacOS). |
| `arch_id` | Target Arch ID (e.g., `0x01` for x86_64, `0x02` for ARM64). |
| `relative_path` | Path to the artifact relative to the lockfile. |

**Rules:**
- Multiple `@artifact` directives can exist for the same `content_id` if a package provides multiple platform binaries.
- Artifacts are validated against the `Artifacts` array in the binary payload.

### 3.5 Patch Directive (Unchanged)
```text
@patch <content_id_hex> <patch_type> <relative_path>
```

## 4. Content IDs
Unchanged. FNV-1a 64-bit hashes of `canonical_name@major.minor.patch`.

## 5. The Binary Payload (v0.11 Schema)

### 5.1 Payload Layout
The version byte is now `0x07`. The layout is strictly sequential. Auto-detection heuristics are gone.

| Field | Type | Size | Description |
|---|---|---|---|
| `Version` | Byte | 1 byte | `0x07` |
| `LogicalNameLen` | Varint | 1+ bytes | Length of logical name, or `0x00` if none. |
| `LogicalName` | Bytes | Variable | The alias name. |
| `SourceIdx` | Varint | 1+ bytes | Index into the `@source` array. |
| `Major` | Varint | 1-9 bytes | |
| `Minor` | Varint | 1-9 bytes | |
| `Patch` | Varint | 1-9 bytes | |
| `HashCount` | Varint | 1+ bytes | |
| `Hashes` | Array | Variable | Integrity hashes of the package source. |
| `FeatureCount` | Varint | 1+ bytes | |
| `Features` | Array | Variable | Enabled feature flags. |
| `PeerCount` | Varint | 1+ bytes | |
| `Peers` | Array | Variable | Resolved peer dependencies. |
| `DepCount` | Varint | 1+ bytes | |
| `Dependencies` | Array | Variable | Standard dependencies. |
| `PeerReqCount` | Varint | 1+ bytes | |
| `PeerRequirements` | Array | Variable | Unresolved peer requirements. |
| `PlatformTagCount`| Varint | 1+ bytes | |
| `PlatformTags` | Array | Variable | Target platform constraints. |
| `ExportCount` | Varint | 1+ bytes | **NEW** |
| `Exports` | Array | Variable | **NEW** (see §5.2) |
| `ArtifactCount` | Varint | 1+ bytes | **NEW** |
| `Artifacts` | Array | Variable | **NEW** (see §5.3) |
| `HookHashCount` | Varint | 1+ bytes | Renamed from `ScriptHashCount`. |
| `HookHashes` | Array | Variable | Renamed from `ScriptHashes` (see §5.4). |
| `PatchHash` | Bytes | 0 or 33 bytes | |
| `CRC32` | Bytes | 4 bytes LE | Checksum of all preceding bytes. |

### 5.2 Export Struct Layout
Prevents "dependency confusion" attacks by locking the public interface of the module. 

| Field | Type | Size | Description |
|---|---|---|---|
| `IdentifierLen` | Varint | 1+ bytes | Length of the export name (e.g., `./core`, `Main`). |
| `Identifier` | Bytes | Variable | The public interface name. |
| `HashAlgo` | Byte | 1 byte | Algorithm used to hash the resolved interface. |
| `DigestLen` | Byte | 1 byte | Length of the digest. |
| `Digest` | Bytes | Variable | Hash of the resolved file/module behind the interface. |

**Semantics:**
- If a module defines an export map in its manifest, *all* entries SHOULD be listed here.
- The digest is computed over the absolute canonical path or the content hash of the resolved file, NOT the manifest string itself.
- If a consumer requests an import that is NOT in this list, the resolver SHOULD error (preventing phantom exports).

### 5.3 Artifact Struct Layout
Secures dynamically fetched or compiled binaries (e.g., Rust binaries distributed via a generic build tool, WASM modules).

| Field | Type | Size | Description |
|---|---|---|---|
| `OsId` | Byte | 1 byte | Target OS ID. |
| `ArchId` | Byte | 1 byte | Target Arch ID. |
| `HashAlgo` | Byte | 1 byte | Hash algorithm ID. |
| `DigestLen` | Byte | 1 byte | Length of the digest. |
| `Digest` | Bytes | Variable | Hash of the artifact file on disk. |

**Semantics:**
- Artifacts listed here MUST have a corresponding `@artifact` header directive.
- Installers MUST verify the file at `relative_path` matches this digest before allowing the build tool to use it.

### 5.4 Hook Hash Struct Layout (Renamed from ScriptHash)
Generic lifecycle hooks (e.g., `pre-build`, `post-compile`, `test-setup`).

| Field | Type | Size | Description |
|---|---|---|---|
| `HookTypeLen` | Varint | 1+ bytes | **CHANGED** (was a fixed byte). |
| `HookType` | Bytes | Variable | **CHANGED** Raw string of the hook name (e.g., `"post-compile"`). |
| `HashAlgo` | Byte | 1 byte | Hash algorithm ID. |
| `DigestLen` | Byte | 1 byte | Length of the digest. |
| `Digest` | Bytes | Variable | Hash of the raw hook command/text. |

*Note: We switched from a hardcoded enum byte to a variable-length string to ensure absolute language agnosticism.*

## 6. Public API Specification

### 6.1 Removed API
- `enum CompatMode` (Deleted)
- `fn serialize_compat(...)` (Deleted)
- `fn pack_payload_v8(...)` (Deleted)
- `fn pack_payload_v9(...)` (Deleted)

### 6.2 New Data Structures
```rust
#[derive(Debug, Clone, PartialEq, Eq)]
pub struct Export {
    pub identifier: String,
    pub hash_algo: HashAlgorithm,
    pub digest: Vec<u8>,
}

#[derive(Debug, Clone, PartialEq, Eq)]
pub struct Artifact {
    pub os_id: u8,
    pub arch_id: u8,
    pub hash_algo: HashAlgorithm,
    pub digest: Vec<u8>,
}

#[derive(Debug, Clone, PartialEq, Eq)]
pub struct HookHash {
    pub hook_type: String, // Changed from enum ScriptType
    pub hash_algo: HashAlgorithm,
    pub digest: Vec<u8>,
}

#[derive(Debug, Clone, PartialEq, Eq)]
pub struct ArtifactDirective {
    pub content_id: u64,
    pub os_id: u8,
    pub arch_id: u8,
    pub relative_path: String,
}
```

```rust

pub struct Package {
```

### 6.4 New Error Variants
```rust
#[error("Unrecognized export '{identifier}' requested from package '{package}'")]
    UnknownExport { package: String, identifier: String },

#[error("Artifact for '{package}' (os: {os}, arch: {arch}) not found at '{path}'")]
    ArtifactMissing { package: String, os: u8, arch: u8, path: String },

#[error("Artifact for '{package}' digest mismatch: expected {expected}, got {actual}")]
    ArtifactDigestMismatch { package: String, expected: String, actual: String },
```

## 7. Implementation Notes

### 7.1 The Strict Parser
The v0.11 parser is significantly simpler than v0.10. 
```rust
// Old v0.10 logic (removed):
let is_v9 = cursor + 4 < bytes.len();
let is_v10 = cursor + 4 < bytes.len();

// New v0.11 logic:
assert_eq!(bytes[0], 0x07);
// Just read fields sequentially. If it fails, the file is corrupt.
let export_count = read_varint(...);
for _ in 0..export_count { ... }
```

### 7.2 Dependency Confusion Mitigation
When a build tool resolves an import (e.g., `import { Foo } from "bar/core"`):
1. Resolve `bar` to a package via Content ID.
2. Look up `"bar/core"` in the `Exports` array.
3. If missing, throw `UnknownExport`. Do *not* fall back to filesystem probing.

### 7.3 Artifact Verification
Before executing any build steps:
1. Iterate over `@artifact` directives.
2. Read the file at `relative_path`.
3. Hash it.
4. Compare against the matching `Artifact` struct in the package's payload (matched by `os_id` and `arch_id`).
5. If mismatch, throw `ArtifactDigestMismatch`.

## 8. Migration Guide (v0.10 → v0.11)

There isn't one. This is a hard break.

- **Parsers:** Delete all `if version == 0x06` branching. Reject `0x06` payloads entirely with `Error::UnknownPayloadVersion`.
- **Serializers:** Always serialize as `0x07`. Always include the `Exports` and `Artifacts` arrays (even if empty).
- **Hook Names:** If migrating from a JS-specific tool, map hardcoded JS script names to their raw string equivalents (e.g., `ScriptType::PostInstall` becomes `HookHash { hook_type: "postinstall".to_string(), ... }`).
