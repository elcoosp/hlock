# HLOCK Specification v0.4.0

## 1. Overview
HLOCK v0.4.0 introduces native **Workspace (Monorepo) Support** and **Multi-Algorithm Integrity Hashes**. 

As modern development shifts heavily towards monorepos, lockfiles must differentiate between packages fetched from remote registries (which require integrity hashes) and local workspace packages (which are compiled from source and do not). Furthermore, supply chain best practices now dictate verifying packages against multiple cryptographic hashes (e.g., SHA-256 for the tarball, BLAKE3 for the extracted contents).

**Breaking Change:** HLOCK v0.4.0 parsers will intentionally reject v0.3.0, v0.2.0, and v0.1.0 payloads due to fundamental changes in the binary hash schema. There is no backward compatibility.

## 2. File Structure
Unchanged from v0.3.0. The file is UTF-8 encoded, LF line endings (`\n`), split into a Metadata Header and a Package Graph.

```text
<header directives>
<empty line>
<package lines>
```

### 2.1 The Metadata Header
Unchanged syntax, but introduces a new source type.

*   `@source <idx> workspace`: Defines the local workspace source. Packages pointing to this index are resolved locally rather than fetched remotely.

**Example Header:**
```text
@source 0 https://registry.npmjs.org/
@source 1 workspace
@override react 18.2.0 -> 18.2.1
```

## 3. The Binary Payload (v0.4 Schema)
The binary payload has been modified to replace the single hash array with a dynamic array of typed hashes.

### 3.1 Payload Layout

| Field | Type | Size | Description |
| :--- | :--- | :--- | :--- |
| `Version` | Byte | 1 byte | Payload schema version. **MUST be `0x03`** for v0.4.0. |
| `SourceIdx`| Varint | 1+ bytes | Index of the download source mapping to `@source <idx>`. |
| `Major` | Varint | 1-9 bytes | Package major version. |
| `Minor` | Varint | 1-9 bytes | Package minor version. |
| `Patch` | Varint | 1-9 bytes | Package patch version. |
| `HashCount`| Varint | 1+ bytes | Number of integrity hashes provided for this package. |
| `Hashes` | `HashCount` x Struct | Variable| Array of hash structs (see 3.2). |
| `DepCount`| Varint | 1+ bytes | Number of direct dependencies. |
| `Dependencies`| `DepCount` x Struct | Variable| Array of dependency structs (see 3.3). |
| `CRC32` | Bytes | 4 bytes | Standard CRC32 (IEEE 802.3) checksum of all preceding bytes, little-endian. |

### 3.2 Hash Struct Layout

| Field | Type | Size | Description |
| :--- | :--- | :--- | :--- |
| `HashAlgo`| Byte | 1 byte | Integer representing the hash algorithm (see 3.4). |
| `HashLen` | Byte | 1 byte | Length of the hash digest in bytes. |
| `Hash` | Bytes | `HashLen` bytes | The raw hash digest bytes. |

**Workspace Constraint:** If a package's `SourceIdx` points to a `workspace` source, the parser **MUST** enforce that `HashCount` is exactly `0`. Workspace packages must not have integrity hashes.

### 3.3 Dependency Struct Layout
Unchanged from v0.3.0.

| Field | Type | Size | Description |
| :--- | :--- | :--- | :--- |
| `LineIdx` | Varint | 1+ bytes | 0-based line index of the dependency. |
| `DepType` | Byte | 1 byte | Integer representing the dependency profile (`0x00` to `0x03`). |

### 3.4 Hash Algorithm Enum
*   `0x00`: **SHA-1** (Legacy, 20 bytes)
*   `0x01`: **SHA-256** (Standard, 32 bytes)
*   `0x02`: **SHA-512** (High security, 64 bytes)
*   `0x03`: **BLAKE3** (Modern, 32 bytes)

## 4. Public API Specification

### 4.1 Core Data Structures

```rust
#[derive(Debug, Clone, PartialEq, Eq)]
pub enum Source {
    Registry(String),
    Local(String),
    Git(String),
    Workspace, // NEW
}

#[derive(Debug, Clone, PartialEq, Eq)]
pub enum HashAlgorithm {
    Sha1,
    Sha256,
    Sha512,
    Blake3,
}

// NEW: Replaces raw Vec<u8> hash
#[derive(Debug, Clone)]
pub struct IntegrityHash {
    pub algo: HashAlgorithm,
    pub digest: Vec<u8>,
}

#[derive(Debug, Clone)]
pub struct Package {
    pub name: String,
    pub source_idx: usize,
    pub major: u64,
    pub minor: u64,
    pub patch: u64,
    pub hashes: Vec<IntegrityHash>, // CHANGED: from Vec<u8>
    pub dependencies: Vec<Dependency>,
}
// Lockfile, DepType, Dependency, Override remain unchanged from v0.3.0
```

### 4.2 Error Handling Additions

```rust
pub enum Error {
    // ... existing errors ...
    
    #[error("Line {line_number}: Workspace package cannot have integrity hashes")]
    InvalidWorkspaceHash { line_number: usize },

    #[error("Line {line_number}: Unsupported hash algorithm {algo_id}")]
    UnknownHashAlgorithm { line_number: usize, algo_id: u8 },
}
```

### 4.3 Core Functions
*   `fn serialize(lockfile: &mut Lockfile) -> Result<String, Error>`
    *   Validates that any package pointing to `Source::Workspace` has an empty `hashes` vec.
    *   Encodes `hashes` into the new variable-length struct array.
*   `fn deserialize(content: &str) -> Result<Lockfile, Error>`
    *   Parses the new hash struct array.
    *   Validates workspace hash constraints.
