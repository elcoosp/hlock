  why          Explain why a package is in the lockfile

  deps         List dependencies of a package

### Global Flag Semantics

| Flag | Behavior |
|------|----------|
| `--quiet` | Suppress all stdout. Only emit errors to stderr. Exit codes still reflect results. Overrides `--verbose`. |
| `--verbose` | Print extra diagnostics: parse time, package count during scan, individual check timings in `check`. |
| `--no-color` | Equivalent to `--color=never`. |
| `--color auto` | Colorize if stdout is a TTY, no color otherwise. |
| `--color always` | Always colorize, even when piped. |
| `--color never` | Never colorize. |

All subcommands inherit these flags. The `--format json` flag on individual commands always disables color regardless of `--color`.

## 4. Stdin Support (F10)

### Specification

Every command that accepts a `<FILE>` argument also accepts `-` to read from stdin.

Affected commands: `verify`, `lint`, `diff` (both files), `audit`, `sbom`, `sign`, `graph`, `info`, `why`, `check`, `tree`, `licenses`, `deps`, `dependents`, `dedup`, `merge` (all three files).

### Implementation

Add a helper in `main.rs`:

```rust
fn read_input(path: &Path) -> Result<String, Error> {
    if path == Path::new("-") {
        std::io::read_to_string(std::io::stdin())
            .map_err(|e| Error::Io(e))
    } else {
        std::fs::read_to_string(path)
            .map_err(|e| Error::Io(e))
    }
}
```

Replace every `std::fs::read_to_string(&file)` call with `read_input(&file)`.

For `sign --in-place`, if the input file is `-`, print an error to stderr and exit with code 2 (in-place requires a real file).

For `diff`, both `old_file` and `new_file` independently support `-`.

For `merge`, `--base`, `--ours`, and `--theirs` independently support `-`.

```rust
use owo_colors::OwoColorize;

struct ColorConfig {
    enabled: bool,
}

impl ColorConfig {
    fn from_cli(color_when: &str) -> Self {
        let enabled = match color_when {
            "always" => true,
            "never" => false,
            _ => atty::is(atty::Stream::Stdout), // "auto"
        };
        Self { enabled }

    // If !enabled, styles are no-ops

```

Actually, `owo-colors` supports `.style_if_supported(true)` and `.on_supports_color()`. The simplest approach: use `owo-colors` with the `supports-colors` feature, and gate all coloring behind a `ColorConfig` that checks our `--color` flag. When `--color=never`, set `owo-colors::set_override(false)`. When `--color=always`, set `owo-colors::set_override(true)`. When `auto`, don't override.

```rust
fn configure_colors(color_when: &str) {
    match color_when {
        "always" => owo_colors::set_override(true),
        "never" => owo_colors::set_override(false),
        _ => {} // auto: respect TTY detection
    }
}
```

This is the minimal approach. No `atty` dependency needed — `owo-colors` handles TTY detection internally when no override is set.

When `--format json` is used on any command, call `owo_colors::set_override(false)` before output.

## 6. Bugfixes

### F13: Fix Lint `--rule` Filtering

**Current behavior:** `--rule` is accepted but prints "Named rule filtering not yet supported, using default ruleset" and ignores the argument.

**New behavior:**

`--rule <RULE_NAME>` selects which rules to run. Can be specified multiple times.

`--rule -<RULE_NAME>` excludes a rule (prefix with `-`).

If no `--rule` is given, all default rules run (current behavior).

Available rule names:
- `no-git-urls`
- `require-integrity`
- `no-sha1`
- `no-peer-as-runtime`
- `max-depth`
- `require-attestation`
- `no-known-vulnerabilities`
- `require-license`
- `deny-copyleft`
- `require-trust-root`
- `no-expired-keys`
- `deny-postinstall`

**Implementation:**

```rust
fn build_rule_set(rule_args: &[String]) -> Vec<Box<dyn LintRule>> {
    let all_rules: Vec<(String, Box<dyn LintRule>)> = vec![
        ("no-git-urls".into(), Box::new(NoGitUrls) as Box<dyn LintRule>),
        ("require-integrity".into(), Box::new(RequireIntegrity)),
        ("no-sha1".into(), Box::new(NoSha1)),
        ("no-peer-as-runtime".into(), Box::new(NoPeerAsRuntime)),
        ("max-depth".into(), Box::new(MaxDepth { max: 5 })),
        ("require-attestation".into(), Box::new(RequireAttestation)),
        ("no-known-vulnerabilities".into(), Box::new(NoKnownVulnerabilities)),
        ("require-license".into(), Box::new(RequireLicense)),
        ("deny-copyleft".into(), Box::new(DenyCopyleft)),
        ("require-trust-root".into(), Box::new(RequireTrustRoot)),
        ("no-expired-keys".into(), Box::new(NoExpiredKeys)),
        ("deny-postinstall".into(), Box::new(DenyPostinstall)),
    ];

    let mut includes: HashSet<String> = HashSet::new();
    let mut excludes: HashSet<String> = HashSet::new();

    for arg in rule_args {
        if let Some(name) = arg.strip_prefix('-') {
            excludes.insert(name.to_string());
        } else {
            includes.insert(arg.clone());
        }
    }

    if includes.is_empty() && excludes.is_empty() {
        // Default: all rules
        return all_rules.into_iter().map(|(_, r)| r).collect();
    }

    all_rules.into_iter()
        .filter(|(name, _)| {
            if !includes.is_empty() {
                includes.contains(name)
            } else {
                true
            }
        })
        .filter(|(name, _)| !excludes.contains(name))
        .map(|(_, r)| r)
        .collect()
}
```

If a rule name is not recognized, print error to stderr and exit with code 2.

### F14: Fix Memory Leak in `verify --trusted-key`

**Current behavior:** `Box::leak(pubkey.into_boxed_slice())` leaks memory for each trusted key.

**New behavior:** Store trusted keys as owned `Vec<u8>`. Change `verify_signature` to accept `&[u8]` from the `Vec`.

**Implementation:**

Change the `trusted` map in the `Verify` command handler:

```rust
let mut trusted: HashMap<String, (Vec<u8>, signature::SignatureAlgorithm)> = HashMap::new();
for spec in &trusted_key {
    if let Some((key_id, (pubkey, algo))) = parse_trusted_key(spec) {
        trusted.insert(key_id, (pubkey, algo));
    }
}
```

Update `verify_signature` signature:

```rust
pub fn verify_signature(
    lockfile_content: &str,
    trusted_keys: &HashMap<String, (Vec<u8>, SignatureAlgorithm)>,
) -> Result<(), SignatureError>
```

Inside `verify_signature`, change `expected_pub_key` usage from `&[u8]` to `&Vec<u8>` (auto-deref to `&[u8]` works). The `try_into` conversions for `[u8; 32]` etc. remain unchanged since they operate on slices.

Update `signature.rs` tests to use `HashMap<String, (Vec<u8>, SignatureAlgorithm)>` instead of `&'static [u8]`.

**Implementation:**

```rust
fn render_tree(lockfile: &Lockfile, root: &str, max_depth: Option<u32>, dep_type_filter: &str) -> Result<TreeNode, Error> {
    let root_pkg = lockfile.packages.iter().find(|p| p.name == root)
        .ok_or_else(|| /* package not found */)?;

    fn build_node(lockfile: &Lockfile, pkg: &Package, depth: u32, max_depth: Option<u32>, dep_type_filter: &str, visited: &mut HashSet<String>) -> TreeNode {
        if max_depth.map_or(false, |max| depth >= max) {
            return TreeNode { name: pkg.name.clone(), version: version_string(pkg), deps: vec![] };
        }
        let mut node = TreeNode { name: pkg.name.clone(), version: version_string(pkg), deps: vec![] };
        for dep in &pkg.dependencies {
            // filter by dep_type_filter
            if !visited.contains(&dep.name) {
                visited.insert(dep.name.clone());
                if let Some(dep_pkg) = lockfile.packages.iter().find(|p| p.name == dep.name) {
                    node.deps.push(build_node(lockfile, dep_pkg, depth + 1, max_depth, dep_type_filter, visited));
                }
                visited.remove(&dep.name);
            }
            // if visited, mark as cycle
        }
        node
    }

    let mut visited = HashSet::new();
    visited.insert(root.to_string());
    Ok(build_node(lockfile, root_pkg, 0, max_depth, dep_type_filter, &mut visited))
}
```

Tree text rendering with Unicode box-drawing characters (`├──`, `└──`, `│   `).
